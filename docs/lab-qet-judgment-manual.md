# 尺寸原始紀錄表判定操作說明

本說明適用於以下兩頁：

- `尺寸原始紀錄表判定`
- `尺寸原始紀錄表判定結果`

相關路由：

- `#/lab/qet/judge`
- `#/lab/qet/results`

## 一、功能目的

### 1. 尺寸原始紀錄表判定

用來針對已建立的 `QET-15-01 尺寸原始紀錄表` 進行最終判定。

可執行：

- 查看所有尺寸原始紀錄表
- 選擇判定結果
- 按 `確認` 儲存判定
- 按 `取消` 清除已判定結果
- 點委託單編號開啟原始尺寸紀錄表內容

### 2. 尺寸原始紀錄表判定結果

用來瀏覽已完成判定的記錄表結果。

僅供查閱，不提供修改。

## 二、尺寸原始紀錄表判定頁

### 1. 進入頁面後顯示內容

系統會自動載入尺寸原始紀錄表清單，依 `日期由新到舊` 排列。

欄位說明：

- `委託單編號`
  點擊後可開啟該張尺寸原始紀錄表。
- `客戶名稱`
  來自來源委託單資料。
- `測試件品名`
  對應尺寸原始紀錄表中的品名。
- `日期`
  對應委託單日期或記錄表填表日期。
- `判定`
  下拉選單，可選：
  - `PENDING`
  - `PASS`
  - `FAIL`
- `取消`
  清除已做的判定，恢復成可重新判定狀態。
- `確認`
  將目前選擇的判定結果正式寫入系統。

### 2. 分頁功能

頁面底部提供分頁控制。

目前規則：

- 每頁 `20` 筆
- 顯示：
  - `共幾筆`
  - `目前第幾頁`
  - `總頁數`
- 可用按鈕：
  - `上一頁`
  - `下一頁`

### 3. 如何進行判定

操作步驟：

1. 進入 `尺寸原始紀錄表判定`
2. 找到要判定的委託單
3. 在 `判定` 下拉選單選擇：
   - `PASS`
   - `FAIL`
4. 按 `確認`
5. 系統會跳出確認訊息
6. 確認後，系統會寫入：
   - 判定結果
   - 判定人
   - 判定時間

### 4. 如何取消判定

操作步驟：

1. 在該筆資料按 `取消`
2. 系統會跳出警告確認
3. 確認後，該筆資料會恢復成未完成判定狀態

取消後效果：

- 判定結果恢復為 `PENDING`
- 判定日期清空
- 判定人清空

### 5. 重新整理

頁面右上角有 `重新整理` 按鈕。

使用時機：

- 剛完成判定後想立即更新畫面
- 有其他人同時操作時，想重新抓取最新資料

## 三、尺寸原始紀錄表判定結果頁

### 1. 功能說明

此頁只顯示已完成判定的資料。

用途：

- 查閱歷史判定結果
- 查閱判定日期
- 查閱審查人

### 2. 顯示欄位

- `委託單編號`
- `客戶名稱`
- `測試件品名`
- `委託單日期`
- `判定結果`
- `判定日期`
- `審查人`

### 3. 分頁功能

頁面底部提供分頁控制。

目前規則：

- 每頁 `20` 筆
- 顯示總筆數與頁數
- 可用：
  - `上一頁`
  - `下一頁`

### 4. 開啟原始表單

點 `委託單編號` 可直接開啟該筆對應的尺寸原始紀錄表，方便交叉查看內容。

## 四、判定資料實際儲存內容

系統會將判定資訊寫入 `dbo.lab_qet_form`。

主要欄位：

- `final_result`
  判定結果
- `judge_date`
  判定日期
- `judge_reviewer`
  判定人
- `reviewer`
  審查者

## 五、常見操作情境

### 情境 1：完成尺寸原始紀錄表後進行判定

1. 先完成 `尺寸原始紀錄表`
2. 儲存表單
3. 進入 `尺寸原始紀錄表判定`
4. 找到該筆資料
5. 選擇 `PASS` 或 `FAIL`
6. 按 `確認`

### 情境 2：判定錯誤，需重判

1. 進入 `尺寸原始紀錄表判定`
2. 找到該筆資料
3. 按 `取消`
4. 重新在下拉選單選擇正確結果
5. 按 `確認`

### 情境 3：僅查歷史判定結果

1. 進入 `尺寸原始紀錄表判定結果`
2. 直接透過分頁查閱
3. 若要看內容，點 `委託單編號`

## 六、注意事項

- `判定結果頁` 為唯讀頁面，不可直接修改。
- 若要修改既有判定，請回到 `尺寸原始紀錄表判定` 頁執行 `取消` 後重判。
- 若畫面資料不是最新，請先按 `重新整理`。
- 若點按後出現 `502` 或無法連線，通常代表 backend 未正常啟動，需重新執行 `startup.bat` 或重啟 backend。
